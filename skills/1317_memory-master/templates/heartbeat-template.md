# 💓 心跳检测指南

## 概述

当你收到心跳检测（消息匹配配置的心跳提示）时，不要每次都只回复 `HEARTBEAT_OK`。要有效利用心跳检测！

默认心跳提示：
`如果 HEARTBEAT.md 存在（工作空间上下文），请阅读它。严格遵守。不要推断或重复先前聊天中的旧任务。如果不需要关注任何事，回复 HEARTBEAT_OK。`

你可以自由编辑 `HEARTBEAT.md`，添加简短检查清单或提醒。保持简短以限制token消耗。

## 心跳检测 vs 定时任务：何时使用

**使用心跳检测时：**

- 多个检查可以批量处理（收件箱 + 日历 + 通知一次性完成）
- 你需要来自最近消息的对话上下文
- 时间可以稍微漂移（每~30分钟没问题，不需要精确）
- 你想通过合并定期检查来减少API调用

**使用定时任务时：**

- 精确时间很重要（"每周一上午9:00整"）
- 任务需要与主会话历史隔离
- 你想要为任务使用不同的模型或思考级别
- 一次性提醒（"20分钟后提醒我"）
- 输出应直接发送到频道，不涉及主会话

**提示：** 将类似的定期检查批量放入 `HEARTBEAT.md`，而不是创建多个定时任务。使用定时任务进行精确时间表和独立任务。

## 检查事项（每天轮换2-4次）

- **电子邮件** - 有任何紧急未读消息吗？
- **日历** - 未来24-48小时内有即将发生的事件吗？
- **提及** - Twitter/社交媒体通知？
- **天气** - 如果用户可能外出，相关吗？

## 何时联系用户

- 重要电子邮件到达
- 日历事件即将发生（<2小时）
- 你发现有趣的事情
- 距离你上次说话已经 >8小时

## 何时保持安静（HEARTBEAT_OK）

- 深夜（23:00-08:00）除非紧急
- 用户明显很忙
- 自上次检查以来没有新内容
- 你刚刚在 <30分钟前检查过

## 无需询问即可进行的主动工作

- 阅读和组织记忆文件
- 检查项目（git状态等）
- 更新文档
- 提交并推送你自己的更改
- **审查并更新 MEMORY.md**（见下文）

## 记忆维护（在心跳检测期间）

定期（每隔几天）使用心跳检测：

1. 阅读最近的 `memory/YYYY-MM-DD.md` 文件
2. 识别值得长期保存的重要事件、教训或见解
3. 用精炼的学习内容更新 `MEMORY.md`
4. 从 MEMORY.md 中删除不再相关的过时信息

就像人类回顾日记并更新心理模型一样。每日文件是原始笔记；MEMORY.md 是精心整理的知识。

目标：乐于助人但不烦人。每天检查几次，做有用的后台工作，但尊重安静时间。
