# 重要教训/经验记录

> 根据 memory-master v2.6.0，此文件用于记录从经验中学到的重要教训，而非操作规则。
> 操作规则已迁移至 AGENTS.md。

## 教训记录

### 记忆系统教训
- **规则分离的重要性**：操作规则应放在 AGENTS.md，重要教训放在 MEMORY.md
- **自动写入的必要性**：AI 必须自动记录，不依赖用户提醒
- **结构化格式的价值**："因-改-待"格式使记忆清晰可追溯

### 技术实践教训
- **知识库组织**：按主题大类组织知识，避免过多小文件
- **网络学习流程**：Tavily + web_fetch 组合最有效
- **浏览器自动化**：Playwright 的 click+type 组合最可靠

### 用户交互教训
- **群聊参与度**：质量 > 数量，选择性参与
- **任务分配**：复杂任务交给子代理，主会话保持响应
- **安全边界**：内部操作可直接进行，外部操作需询问

---

**如何记录新教训：**

1. **识别教训**：从错误、成功经验、用户反馈中提取
2. **分类记录**：按主题（记忆系统、技术实践、用户交互等）组织
3. **保持简洁**：用要点形式记录核心教训
4. **定期回顾**：每隔一段时间回顾并更新教训

**示例格式：**
```
### [主题] 教训
- **问题**：描述遇到的问题
- **解决方案**：找到的解决方法  
- **教训**：学到的核心教训
- **应用**：如何应用到未来工作
```
