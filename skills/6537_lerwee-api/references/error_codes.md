| 1000 | 系统错误 | 系统内部错误 | 联系系统管理员 |

| 1004 | 请求方法错误 | HTTP 方法不正确 | 使用 POST 方法 |
| 1005 | 请求格式错误 | 请求体格式不正确 | 确保使用 JSON 格式 |
| 1006 | 缺少必填参数 | 缺少必填的请求参数 | 补充必填参数 |
| 1007 | 参数类型错误 | 参数类型不正确 | 检查参数类型 |
| 1008 | 参数值超出范围 | 参数值不在允许范围内 | 使用合理的参数值 |

| 1011 | IP 限制 | IP 地址不在白名单中 | 联系管理员添加 IP 白名单 |

## 签名验证常见问题

### 问题 1: 签名验证失败 (1002)

**可能原因：**
1. 密钥 (secret) 配置错误
2. 参数拼接顺序错误（必须按字母顺序）
3. 跳过了不该跳过的参数
4. 大小写问题（最终结果必须转小写）
5. 时间戳没有参与签名

**解决方案：**
1. 确认 secret 配置正确
2. 使用 `sorted()` 对参数按键排序
3. 只跳过 `sign` 字段本身、空值和数组/对象类型
4. SHA1 结果使用 `.lower()` 转小写
5. 确保 `timestamp` 参与签名计算

### 问题 2: 参数格式错误 (1001)

**可能原因：**
1. 必填参数缺失
2. 参数类型错误（如字符串传了数字）
3. JSON 格式不正确

**解决方案：**
1. 检查接口文档的必填参数
2. 确认参数类型正确
3. 使用标准 JSON 格式

## 错误处理建议

1. **总是检查 code 字段**：只有 `code == 0` 表示成功
2. **记录 message 内容**：错误信息包含具体失败原因
3. **根据错误码处理**：不同错误码可能需要不同的处理方式
4. **合理重试**：对于网络错误、超时等可以重试，签名错误不应重试
5. **记录日志**：记录错误详情便于排查问题
