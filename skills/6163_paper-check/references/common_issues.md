# 毕业论文常见格式问题与修复方案

## 高频问题 TOP 10

### 1. 首行缩进用固定厘米值而非字符单位

**现象**: 首行缩进使用 `firstLine` (twips) 而非 `firstLineChars`
**影响**: 不同字号下缩进效果不一致，无法精确对应"2字符"要求
**修复**:
```python
ind.set(qn('w:firstLineChars'), '200')   # 200 = 2字符 × 100
ind.set(qn('w:firstLine'), '480')        # 配合值（可选）
```
**严重程度**: 中

---

### 2. 页边距不符合规范

**现象**: 使用 Word 默认的 2.54cm 统一边距
**常见规范**: 上2.8cm/下2.2cm/左3.0cm/右2.0cm
**修复**: 通过 `sectPr > pgMar` 设置正确值
**注意**: 不同节可能需要不同的页边距（如封面页），需逐节检查
**严重程度**: 高

---

### 3. 居中标题继承了正文的首行缩进

**现象**: "摘要"、"目录"等居中标题实际有2字符首行缩进
**原因**: Normal 样式自带首行缩进，居中标题未显式覆盖
**修复**:
```python
# 必须显式设为0，仅删除 <w:ind> 不够
ind.set(qn('w:firstLine'), '0')
ind.set(qn('w:firstLineChars'), '0')
```
**严重程度**: 中

---

### 4. 页眉内容/格式不正确

**现象**:
- 页眉内容不是论文完整题目
- 页眉字体不是规范要求的五号宋体/TNR
- 页眉未居中
- 各节页眉不一致

**修复要点**:
- 遍历所有 section 的 header
- 同时设置 `w:eastAsia` 和 `w:ascii`/`w:hAnsi`
- 注意 linkToPrevious 属性的影响

**严重程度**: 高

---

### 5. 参考文献格式问题

**现象**:
- 标题缺少字间距（应为"参  考  文  献"）
- 正文未使用悬挂缩进
- 文献类型标识错误（如 `[硕士学位论文]` 应为 `[D]`）
- 英文字体不是 Times New Roman

**修复**:
- 标题：替换文本为带全角空格的版本
- 正文：设置悬挂缩进2字符 + 左缩进2字符
- 标识：批量替换文献类型

**严重程度**: 高

---

### 6. 表格未使用三线表

**现象**: 表格有完整边框或不规范的边框样式
**规范**: 三线表 = 上下粗线(1.5磅) + 标题行下细线(1磅) + 无左右线 + 无内部竖线
**修复**:
```python
# 表级：上下粗线，无其他线
# 标题行：通过 tcBorders 添加底部细线
```
**严重程度**: 高

---

### 7. 段前段后间距不当

**现象**:
- 正文段落有多余的段前/段后间距（通常应为0）
- 标题段落缺少规范要求的段前段后间距
- 摘要/致谢标题缺少段前段后1行

**修复**: 分别处理正文和标题段落的间距
**严重程度**: 中

---

### 8. 页眉距/页脚距异常

**现象**: 页眉距和页脚距使用了极小的默认值（如 1.27mm），导致页眉与正文重叠
**常见规范**: 页眉距 1.8cm，页脚距 1.4cm
**修复**: 通过 `pgMar` 的 `header` 和 `footer` 属性设置
**注意**: 不同节的页眉距/页脚距可能需要分别调整
**严重程度**: 中

---

### 9. 装订线设置错误

**现象**: 某些节有非零的装订线宽度，导致正文偏移
**修复**: 将装订线设为 0（除非规范明确要求）
**严重程度**: 低

---

### 10. 行距设置不一致

**现象**: 部分段落行距不是规范要求的 1.5 倍行距
**原因**: 可能使用了固定行距、最小值行距，或继承了不同样式的行距设置
**修复**: 统一设置为 `lineRule="auto"`, `line="360"`（1.5倍=360/240）
**严重程度**: 中

---

## 其他常见问题

### 11. 目录标题有首行缩进
目录标题"目  录"应居中无缩进。

### 12. 英文摘要格式与中文摘要不对应
英文摘要的 Abstract 标题、正文缩进、Key words 格式需单独检查。

### 13. 图题/表题格式
- 图题应在图下方，表题在表上方
- 图题/表题通常五号字，居中
- "图X-Y"和"表X-Y"的编号格式

### 14. 公式编号
- 公式居中，编号靠右
- 编号格式通常为 (X-Y)

### 15. 脚注格式
- python-docx 对脚注支持有限
- 需提醒用户在 Word 中手动检查

## python-docx 已知限制

以下格式项无法通过 python-docx 自动修复：

1. **页眉横线** - 需要操作 `w:pBdr` 中的底部边框，但与样式继承复杂交互
2. **目录更新** - 目录是域代码（Field Code），需在 Word 中按 F9 更新
3. **脚注格式** - python-docx 对脚注/尾注的修改支持有限
4. **域代码** - 页码域、交叉引用等需在 Word 中操作
5. **图片精确位置** - 浮动图片的环绕方式和锚定位置调整困难
6. **SmartArt / 图表** - 嵌入的 Excel 图表和 SmartArt 无法修改
7. **批注和修订** - 建议在 Word 中清理后再进行格式修复

## 修复优先级建议

| 优先级 | 类别 | 说明 |
|--------|------|------|
| P0 | 页面设置 | 影响全局排版，最先修复 |
| P0 | 页眉页脚 | 每页可见，影响大 |
| P1 | 正文首行缩进 | 涉及段落多，批量修复 |
| P1 | 表格三线表 | 格式差异明显 |
| P1 | 参考文献 | 格式要求严格 |
| P2 | 标题间距 | 段前段后设置 |
| P2 | 摘要/致谢 | 特殊部分格式 |
| P3 | 字体检查 | 逐个 run 检查 |
| P3 | 其他 | 图题表题、脚注等 |
